# 產品需求文件 (PRD) - 個人記帳簿

## 1. 專案概述
- **背景與動機**：大學生或一般學生族群常因為沒有穩定收入，且生活開銷細碎，導致不知不覺成為「月光族」。為了解決這個問題，我們需要一個簡單且直覺的記帳系統，幫助學生建立良好的財務觀念，了解每一筆錢的去向。
- **目標用戶**：學生族群（有預算控制需求、希望掌握生活費收支狀況的使用者）。
- **核心價值主張**：記錄收入與支出，顯示餘額統計，幫助用戶輕鬆掌握現金流，達成理財目標。

## 2. 功能需求
以下為系統主要功能及其使用者故事：

### (1) 明確現金流向
- **作為** 學生用戶，**我希望** 能夠記錄每一筆收入和支出的金額、日期與備註，**以便** 我能清楚知道我的錢從哪裡來、花到哪裡去。

### (2) 消費分類與分析
- **作為** 學生用戶，**我希望** 在記帳時可以選擇消費分類（如：餐飲、交通、娛樂、學習等），並查看圖表分析，**以便** 我知道每個月在哪個項目上花費最多。

### (3) 設定每月預算
- **作為** 學生用戶，**我希望** 能為不同分類設定每個月的預算，當花費接近預算上限時能有提示，**以便** 我能控制開銷，避免超支。

### (4) 淨資產資訊
- **作為** 學生用戶，**我希望** 系統能統整我的總資產與總負債，並計算出目前的淨資產，**以便** 我了解當下整體的財務健康狀況。

### (5) 多帳戶管理
- **作為** 學生用戶，**我希望** 能建立多個資金帳戶（如：現金、悠遊卡、郵局帳戶、銀行帳戶等），並能在帳戶間進行轉帳記錄，**以便** 我的記帳能更貼近實際的資產分佈。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask (Python)
  - 前端模板：Jinja2 + HTML/CSS/JS
  - 資料庫：SQLite
- **效能考量**：
  - 系統應在 1 秒內載入主要儀表板與圖表，確保記帳時的流暢體驗。
  - 設計應支援行動裝置瀏覽（Responsive Web Design），方便學生隨時隨地拿出手機記帳。
- **安全考量**：
  - 使用者密碼需進行雜湊加密（如使用 bcrypt 或 Werkzeug.security）後存入資料庫。
  - 防止常見 Web 攻擊，如 SQL Injection 與 XSS。

## 4. MVP 範圍

| 功能 | 優先級別 | 說明 |
| --- | --- | --- |
| 基礎收支記錄 | Must Have | 新增、修改、刪除日常收入與支出。 |
| 消費分類與圖表分析 | Must Have | 支援基本圓餅圖或長條圖顯示當月各分類支出。 |
| 多帳戶管理 | Should Have | 支援新增不同帳戶並分別計算各帳戶餘額。 |
| 淨資產資訊 | Should Have | 在首頁儀表板統整所有帳戶的總餘額與淨資產。 |
| 設定每月預算 | Nice to Have | 可在設定頁面填寫各分類預算並於儀表板顯示進度條。 |
